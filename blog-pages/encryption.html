Definition:

The definition for Encryption is pretty simple, it is just a process of Encoding information. During encryption, the plain data is converted into an alternate form which can be Decoded using a key when we need it. Encryption is existing nearly from 1900 BC. During 20th Century, Encryption was mostly used in sending messages between Military people but in Today’s world, Encryption is mainly used in the transfer of communication over the Internet for security and commerce.

Types of Encryption Algorithm:

Symmetric Encryption Algorithm:

In Symmetric Encryption Algorithm, both the Encryption and Decryption keys are Same. Let me explain Symmetric Encryption. When we use symmetric encryption algorithm, the data is converted to form that cannot be understood by anyone. Once the encryption is complete, you will have a key to decrypt that data. If you some wants to view the data, you can just provide the key to them and they can decrypt the data. The Drawbacks of the Symmetric Algorithm are obvious, You need to save the key in a protected place to prevent the data from leaked, if you are careless about the key then there would be no point in encrypting the data.

Asymmetric Encryption Algorithm:

Asymmetric Encryption uses two keys for Encryption which are Public Key and Private Key. Public Key is used to encrypt the data where Private key is used for decryption. So How did this Encryption overcome all the disadvantages in Symmetric Encryption?

Asymmetric Encryption Algorithm uses a key generation protocol [a kind of mathematical function] to generate a key pair. Both the keys are Mathematically Connected with each other. This relationship between the keys differs from one algorithm to another. The algorithm is basically a combination of two functions – encryption function and decryption function. To state the obvious, the encryption function encrypts the data and decryption function decrypts it.

Encryption in your Daily Life:

If you observe carefully in the URL Pane while visiting websites, sometimes websites load with http as prefix but sometimes they have https. Now you are reading this blog in a https website, it means your browser established a asymmetrical encryption with our website. Your browser derives the public key of the SSL/TLS certificates installed on our website. So, this key encrypts your information and send to our website during the initial handshake and the website’s private key will decrypt that information. So, your data is safe with us. No intruder can access your data without the private key. Remember that if you are using asymmetric encryption, Never Share Your Private Key.
