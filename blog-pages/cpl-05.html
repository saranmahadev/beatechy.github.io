Object

Object means an entity, that has state and behavior is known as object. It can be defined as an instance of class. It also contains address and takes some space in memory. It can communicate without taking any details of each other data or a code. The only thing is that it takes the messages and it send a response.

Object can contains three characteristics:  State( represents data value of an object)
 Behavior (functionality of an object i.e.deposits and withdraw)
 Identity(having some unique ID which is not visible to the external user, but used internally to identify internally by each object uniquely)

CLASS



Simply class is defined as the collection of object, it doesn’t take any space memory like an object. It consists of data members and methods. The main function of class is to hold data. The member function determines the behavior of the class. Here you can make a blueprint from which you can create an individual object.

INHERITANCE



Defined as the procedure or mechanism of acquiring all the properties and behavior of one class to another. Simply it can be explained as the properties and behavior of child class from the parent class. That is a one object can takes and process all the properties and behavior of another one. And it provides reusability and establishes relationship between different class. A class which inherits the property is known as child class(sub class or derived class). The class which s responsible for carrying its properties is known as parent class(Super class or base class)

POLYMORPHISM(one interface-multiple implementations)



A a certain task can be done through in different methodical ways I.e. polymorphism. In another way it can be defined as it has the ability to create a property, a function, or an object that has more than one realization. It has the ability to substitute classes that have common functionality in sense of methods and data. In polymorphism, the member function with same name are defined in base class and also in each derived class. It also used tp keep the interface of base class to its derived classes. Polymorphism can be achieved by means of virtual functions.

Polymorphism is classified into two ways:


# Method Overloading (compile time polymorphism)

It is a features to allows a class to have two or more methods having the same name but the arguments passed to the Methods are different. Compiling time of polymorphism refers to a process in which a call to an overload method is resolved at compile time rather than at run time

# Methods Overriding(Run Time Polymorphism)

If sub-class(child class) has the same methods as declared in the parent class, it is called as method overriding. If subclass provides the provides the specific implementation of the method that has been provided by one off its parent class, it is known as method overriding.

ABSTRACTION



It is the process of hiding the implementation details and showing only functionality to the user. For an example if you use a computer it provides only the output but it doesn’t shows the the internal processing.which means you only see the output you don’t know the internal processing.

ENCAPSULATION(Information hiding)



It is used to combine data members and functions is called as a class. This is used to prevent then data to access directly.the access is provide to them only the class. Encapsulation is one of the familiar features in Object-Oriented programming in the concept of data hiding. So, it is also called as information hiding.
